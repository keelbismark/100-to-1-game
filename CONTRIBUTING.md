# Руководство по внесению вклада

Спасибо за интерес к проекту "Сто к Одному"! Это руководство поможет вам начать contributing.

## Как начать

### 1. Форкните репозиторий

Нажмите кнопку "Fork" в правом верхнем углу страницы GitHub.

### 2. Клонируйте ваш форк

```bash
git clone https://github.com/ваше-имя/100-to-1.git
cd 100-to-1
```

### 3. Создайте ветку для вашей функции

```bash
git checkout -b feature/ваша-функция
```

### 4. Установите зависимости

```bash
npm install:all
```

### 5. Запустите проект в режиме разработки

Откройте два терминала:

Терминал 1 (сервер):
```bash
cd server
npm run dev
```

Терминал 2 (клиент):
```bash
cd client
npm run dev
```

## Разработка

### Структура кода

```
.
├── server/                      # Backend
│   └── src/
│       └── server.js           # Socket.io логика
├── client/                      # Frontend
│   └── src/
│       ├── components/          # React компоненты
│       │   ├── Board/         # Табло
│       │   ├── Admin/         # Панель ведущего
│       │   ├── Buzzer/        # Кнопка игрока
│       │   └── BigGame.jsx    # Большая игра
│       ├── App.jsx            # Главный компонент
│       └── main.jsx           # Точка входа
└── game-packs/                # Вопросы в JSON
```

### Конвенции

#### Frontend
- Используйте функциональные компоненты с hooks
- Следуйте принципам React - компоненты должны быть маленькими и переиспользуемыми
- Используйте CSS модули или styled components для стилей
- Все асинхронные операции должны иметь обработку ошибок

#### Backend
- Следуйте принципам REST (для HTTP) и Socket.io (для WebSocket)
- Валидируйте все входные данные
- Логируйте важные события
- Обрабатывайте ошибки и возвращайте понятные сообщения

#### Общее
- Пишите понятные комментарии для сложной логики
- Используйте осмысленные имена переменных и функций
- Следуйте формату кода (Prettier/ESLint)

### Тестирование

Перед тем как отправить PR:

1. Протестируйте изменения вручную
2. Проверьте на разных браузерах (Chrome, Firefox, Safari)
3. Проверьте на мобильных устройствах
4. Убедитесь, что WebSocket работает корректно

### Коммиты

Используйте понятные сообщения коммитов:

```
feat: добавить кнопку сброса игры
fix: исправить баг с таймером
docs: обновить README
style: отформатировать код
refactor: оптимизировать компонент Board
test: добавить тесты для API
```

### Pull Request

1. Убедитесь, что ваш код соответствует стандартам
2. Обновите документацию, если нужно
3. Добавьте описание ваших изменений
4. Укажите какие issues закрывает ваш PR (#<issue-number>)
5. Проверьте что CI проходит
