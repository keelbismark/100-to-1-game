# Обзор проекта "Сто к Одному"

## 🎮 Что это такое?

Полнофункциональное SPA-приложение для проведения популярной игры "100 к 1" (аналог Family Feud) с поддержкой нескольких устройств в реальном времени.

## ✨ Ключевые особенности

### 🏗️ Архитектура
- **Backend**: Node.js + Express + Socket.io
- **Frontend**: React 18 + Vite
- **Протокол**: WebSocket для мгновенной синхронизации (<100мс)
- **Хранение**: В памяти (Redis-ready) + JSON файлы для вопросов

### 🎯 Основные функции

#### 1. Многопользовательская игра
- 📺 **Табло** - Большой экран для отображения игры
- 👨‍💼 **Ведущий** - Панель управления для ведущего
- 🔴 **Кнопка** - Смартфон для игроков
- ⚡ Реальное время через WebSocket

#### 2. Игровые режимы
- 🎯 Простой раунд
- ⚡ Двойной раунд (x2 очки)
- 🔥 Тройной раунд (x3 очки)
- 🔄 Игра наоборот
- 🏆 Большая игра (финал)

#### 3. Управление игрой
- Открывание ответов с анимацией
- Промахи с подсчетом
- Переключение команд
- Начисление очков
- Управление звуком

#### 4. Дополнительные функции
- 📝 Редактор пакетов вопросов
- 🔊 Звуковые эффекты (Howler.js)
- 📱 Адаптивный дизайн
- 🌙 Предотвращение засыпания экрана
- 💾 Сохранение состояния
- 🎨 Красивый UI/UX

## 🚀 Быстрый старт

### 1. Установка зависимостей
```bash
npm install:all
```

### 2. Запуск
**Windows:**
```bash
start.bat
```

**Unix (Linux/macOS):**
```bash
bash start.sh
```

**Или вручную:**
```bash
# Terminal 1 - Server
cd server && npm start

# Terminal 2 - Client  
cd client && npm run dev
```

### 3. Использование
- Откройте `http://localhost:3000`
- Создайте игру
- Откройте соответствующие URL на разных устройствах

### Backend

- **CORS** - Разрешение кросс-доменных запросов
- **UUID** - Генерация уникальных ID

## 📊 WebSocket события

### Клиент → Сервер
- `create_room` - Создание комнаты
- `join_room` - Присоединение к комнате
- `reveal_answer` - Открытие ответа
- `add_mistake` - Добавление промаха
- `switch_team` - Переключение команды
- `award_points` - Начисление очков
- `load_round` - Загрузка раунда
- `buzzer_press` - Нажатие кнопки
- `reset_buzzer` - Сброс кнопки
- `toggle_sound` - Переключение звука

### Сервер → Клиент
- `room_created` - Комната создана
- `joined_room` - Успешное соединение
- `room_updated` - Обновление состояния
- `answer_revealed` - Ответ открыт
- `mistake_added` - Промах добавлен
- `team_switched` - Команда переключена
- `points_awarded` - Очки начислены
- `round_loaded` - Раунд загружен
- `buzzer_pressed` - Кнопка нажата
- `buzzer_reset` - Кнопка сброшена
- `sound_toggled` - Звук переключен
- `error` - Ошибка

## 📝 Создание игровых пакетов

### Через редактор
1. Нажмите "Создать пакет" на главном экране
2. Заполните вопросы и ответы
3. Сохраните как JSON
4. Загрузите в папку `game-packs/`

### Вручную
Следуйте формату в `game-packs/pack1.json`:
```json
{
  "pack_id": "unique_id",
  "title": "Название",
  "rounds": {
    "simple": {
      "question": "Вопрос?",
      "answers": [
        {"text": "Ответ 1", "score": 30},
        // ...
      ]
    }
    // ...
  }
}
```
